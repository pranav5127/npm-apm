# Network Peformance Monitoring (NPM).

- Network Peformance Monitoring (NPM) is the practice of visualizing, monitoring, optimizing, troubleshooting, and reporting on the health, availability, and performance of your network, as experienced by your users.

- NPM ingests telemetry from multiple layers of the network and IT infrastructure.
    - Device Metrics: Info from SNMP, WMI, CLI, APIs, logs etc.
    - Network Flow Data: Traffic records such as NetFlow, JFlow, sFlow, IPFIX.
    - Packet Data: Info from network packets.
    - End Point Analysis

## SNMP Monitoring
- Uses SNMP to track device health, bandwidth, latency, and resource utilization across network infrastructure.

## Flow Data Analysis: 
- Analyzes network traffice flows to understand communication patterns, bandwidt usagem and application performance.

## Full Packet Capture:
- Captures and inspects every network packet for deep troubleshooting, security analysis, and forensic investigation.

## Endpoint Packet Analysis:
- Monitors network performance directly from user devices to provide visibility into remote cloud and encrypted traffic.
