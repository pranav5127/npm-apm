# Application Performance Monitoring (APM)

- APM is the practice of tracking key software application peformance metrics using monitoring software and telemetry data.

- It provides insights into how users experience applications and where performance gaps lie.

- It enables real-time monitoring of applications, capturing telemetry data and metrics such as response time, latency, resource usage, and error rates.

- APM tools help in issue diagnosis and troubleshooting by providing visibility into application components, dependencies and transcations.

## Observability vs Monitoring

- APM is about monitoring application performance and system helath by capturing and displaying data which can be analyzed by various teams.

- Observability checks the system's internal state based on the data generated such logs, metrics and traces.

### Use Cases

#### Runtime Application Architecture 

- This component provides a comprehensive understanding of an application's structure and how its various components interact with each other. 

- It offers visibility for application stack such as servers, databases, APIs third-party services, etc.


#### Real User Monitoring (RUM)

- It focuses on capturing and analyzing user interactions with the applications in real-time.

- It provides insights into the user experience by measuring factors like page load times, transaction duration, overall performance, etc, from the end-user's perspective.

#### Synthetic Monitoring

- It is a component of APM that involves creating simulated user interactions with an applicaton.

- These interactions replicate real user behavior and are used proactively to test and monitor the application's performance.

#### Agentless Monitoring

- Agentless Monitoring in APM gathers performance data without the need for installing agents or modifying application code.

#### Component Monitoring

- It focuses on tracking the individual components and services of an application.

- It involves monitoring the performance, health and availability fo each compoenent.

## Key Metrics of APM

- **Response time**: The duration an application takes to respond to a user’s action.
- **Error rate**: The occurrence frequency of errors within the application.
- **Request rate**: The volume of requests an application receives per unit of time.
- **Application availability (uptime)**: The total operational time during which the appli**cation is acc**essible.
- **CPU usage**: The amount of processing power consumed by the application.
- **Latency**: Latency is the delay in data transmission between a source and destination, measuring the time taken for a user’s request to reach the server and return a response.
- **Application performance index (Apdex) score**: A measurement of the app’s performance relative to predefined “normal” performance standards.
- **Service level agreement (SLA) score**: A metric defining the expected service quality, calculated using response time, error rate, uptime, and request volume over a specific period.
