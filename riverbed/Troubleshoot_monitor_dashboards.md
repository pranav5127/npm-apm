# Using Monitor Dashboards

### Monitoring Vs Troubleshooting 

| Aspect             | Monitoring                                         | Troubleshooting                            |
| ------------------ | -------------------------------------------------- | ------------------------------------------ |
| Goal               | Continuously observe system health and performance | Investigate and resolve a specific problem |
| Timing             | Ongoing, proactive                                 | Reactive (usually after an alert or issue) |
| Focus              | Detect anomalies, trends, and degradation          | Find the root cause of an issue            |
| Output             | Dashboards, metrics, alerts, reports               | Diagnosis, analysis, corrective actions    |
| Questions Answered | "Is everything healthy?"                           | "Why is this broken or slow?"              |


## Enterprise summary dashboard
- Provides a high level overview of health and performance.
- Group data by various location attributes.
- Pinpoint common health issues.

## Monitor application dashboard
- Overview of health, performance, and usage.
- Identify users, locations and activities of concern.
- Identify improvements or degradation of performance.

## Monitor location dashboard
- Enterprise summary dashboard, filtered by location.
- Review device and application performance in one branch, office etc.

## User Experience dashboard
- Reacent activities with the current and historic performance.
- Performance trends.
- User experience per application and device.

### Activity resource analyisis
- Correlate slow activites with high resource utilization such as CPU or Memory.

## Monitor SLA dashboard
- Track internal and external SLA compliance


## NOC dashboard
- Track best and worst performing applications and locations.
- See live changes in usage and remote latency.


## Sentiment survey dashboard
- Provide various details about survey.

# Using Troubleshoot dashboards
- **Summary** dashboards such as Incidents, System or Hardware and Virtual Desktop helps in resolving incidents or widespread issues.
- **Focused** dashboards such as Application, User or Device adn IT Service Desk lets us deep dive on one impacted element to investigate issues.

### Incidents
- A list of active and historical Aternity Incidents
- Find patterns and trends such as (Performance incidents = Activity slowness)
- Anomaly Detection incidents such as increase in application crashes.

### Application
- Helps in indentifing wrost performing activites and other key breakdowns.

### User or Device
- Check overall device status, and whether it's impacted by larger issues.

### Virtual Desktop
- Investigate excessive remote display latency.


### IT Service desk
- Assess device status, specifications and user information.
